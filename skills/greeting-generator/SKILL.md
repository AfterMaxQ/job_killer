# 招呼语生成

## 概述

自动读取候选人的简历和个人信息文件，分析岗位 JD 要求与候选人背景的匹配度，生成约 70 字的招聘平台招呼语。核心约束：**前 18 个字必须极其简洁，直接展示与岗位匹配的最大个人优势**（Boss直聘等平台的消息预览只显示前 18 字左右，决定 HR 是否点开查看）。

## 适用场景

- 用户在招聘平台看到心仪岗位，需要发送第一条消息给 HR
- 用户粘贴岗位 JD，要求生成针对性的招呼语
- 用户提到 Boss直聘、猎聘、智联招聘、前程无忧等平台的私信/招呼功能

## 工作流程

### 第一步：读取候选人信息

必须读取以下文件，全面了解候选人背景：

1. **必读**：`个人信息/` 文件夹下所有 .md 文件（由 init-personal-info 初始化）
2. **必读**：项目根目录下的 `CLAUDE.md`（如有项目说明）
3. **参考**：`temp_resume/` 目录下的 Markdown 简历（如已有，了解不同方向的简历侧重点）

### 第二步：提取 JD 关键要求

从用户提供的岗位描述中提取：

- **硬性要求**：学历、技术栈、工作经验年限、语言能力
- **加分项**：特定项目经验、行业背景、工具使用
- **隐含需求**：岗位名称暗示的核心能力（如"后端开发"→架构能力、"数据分析"→建模能力）

### 第三步：匹配分析

将 JD 要求与候选人背景逐一比对，找出：

- **最强匹配点**：候选人最突出的、与 JD 最相关的 1 个优势 → 用于前 18 字
- **补充匹配点**：2-3 个相关经验/技能 → 用于后 52 字
- **差异化亮点**：候选人有但 JD 未明确要求、却能加分的独特优势

### 第四步：生成招呼语

#### 输出格式（必须严格遵守）

```
[前 18 字：一句话点明最大匹配优势][后续 ~52 字：补充相关经验与求职意向][总字数：≤80 字，目标 ~70 字]
```

#### 前 18 字规则

- **直击要害**：用一句话说清楚"我为什么适合这个岗位"
- **包含关键信息**：如技术栈、经验年限、项目类型、学历亮点等
- **不浪费字数在客套话上**：不要"您好"、"我在xxx看到"、"我对您的岗位很感兴趣"
- **可用顿号并列两个最强关键词**：如"3年Java、Spring Boot实战"

#### 后续 ~52 字规则

- 补充前 18 字未覆盖的匹配信息
- 自然收尾，表达沟通意愿
- 可以用 1-2 个具体数据或项目名增加可信度

#### 总字数规则

- 目标 70 字（±10 字可接受，即 60-80 字）
- 如果匹配点太多，优先保留与 JD 最相关的
- 宁缺毋滥：不要为了凑字数加空洞的形容词

## 多简历版本处理

候选人可能有多份不同方向的简历（如 Java 后端、数据分析、机器学习）。当 JD 方向明确时：

1. 优先读取 `个人信息/`（含全部信息）
2. 关注与 JD 方向匹配的经历章节，生成该方向的招呼语
3. 如果用户指定了简历版本，优先使用对应版本的信息

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 前 18 字放客套话（"您好，我对贵公司..."） | 前 18 字直接展示核心竞争力 |
| 招呼语太长，超过 100 字 | 控制在 60-80 字，力求出彩而非堆砌 |
| 泛泛而谈，未针对 JD 定制 | 必须从 JD 中提取关键词并针对性匹配 |
| 编造候选人没有的经验 | 只能使用 `个人信息/` 中实际存在的信息 |
| 语气过于正式或过于随意 | 保持专业、自信、不卑不亢 |

## 生成后检查清单

招呼语生成后，自检以下项目：

- [ ] 招呼语中所有技能/经验是否可追溯到 `个人信息/`？（不编造）
- [ ] 前 18 字是否展示了与 JD 最匹配的优势？
- [ ] 总字数是否在 60-80 字之间？
- [ ] 提到的技能/经验是否均来自简历文件？
- [ ] 是否避免了空洞的形容词（如"学习能力强"、"工作认真"）？
- [ ] 读起来是否自然、像人说的话（不是生成的模板）？
