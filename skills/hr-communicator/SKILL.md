# HR 沟通话术生成

## 概述

HR 回复后，帮用户生成自然、有策略的回复话术。每条话术融入用户个人背景，根据 HR 态度灵活调整语气——热则温，冷则钩，急则准。

**核心哲学**：回复 HR 不是答题，是继续对话。每条回复必须给 HR 一个继续聊的理由。大道至简，回答必须简洁、有逻辑

## 适用场景

- HR 在 Boss 直聘/猎聘/智联等平台回复了，需要回复话术
- 不知道怎么回 HR 的问题（薪资、离职原因、空窗期等）
- 需要多版本话术对比选择
- 需要对 HR 的冷淡态度做"破冰"回复

**不要用于**：首次主动触达 HR（用 `greeting-generator`）、生成简历（用 `build-resume`）、面试准备（用 `interview-question-generator`）。

## 核心流程

```
HR消息 → Step 1: 判断场景 + 检测HR态度
         Step 2: 读取个人信息/ 相关部分
         Step 3: 结合 JD（如有）定位匹配点
         Step 4: 生成 3 版本话术 + 策略说明
```

### Step 1：判断场景 + 检测 HR 态度

**场景判断** — 从 HR 消息中识别属于以下哪种（见下方八类场景详解）。如果 HR 消息涉及多个场景（如同时约面试+问薪资），分别处理。

**态度检测** — 根据 HR 的措辞判断：

| 态度 | 特征 | HR 典型说法 | 你的语气 |
|------|------|-----------|---------|
| 主动 | 热情追问、用感叹号/表情、主动给信息 | "你什么时候方便聊聊？😊" | 从容、展示诚意但不过度热情 |
| 中性 | 礼貌但公事公办、无多余信息 | "请提供期望薪资" | 专业、简洁、有钩子 |
| 冷淡 | 短句、不追问、像在打发 | "嗯""好的""收到" | 抛价值点拉回对话，给对方一个继续的理由 |
| 急切 | 催时间、强调紧急 | "今天能确认吗？""你最快什么时候？" | 精准、不拖泥带水、给确定感 |

### Step 2：读取个人信息

只读相关部分，不全量加载：

| 场景 | 读取的个人信息 |
|------|-------------|
| 约面试 | `基本信息.md`（期望城市、当前身份、时间可用性） |
| 问薪资 | `基本信息.md`（期望薪资范围，如有） |
| 问离职/动机 | `工作经历.md` + `自我评价.md` |
| 问空窗期 | `自我评价.md` + `项目经历/项目列表.md`（空窗期项目/学习） |
| 要材料 | `项目列表.md`（挑最匹配 JD 的 1-2 个） |
| 面试确认 | `基本信息.md`（时间可用性） |
| Offer 沟通 | `基本信息.md`（期望薪资、期望城市） + `工作经历.md` |
| 通用/婉拒 | `自我评价.md` |

**读取原则：**
- 如果 `个人信息/` 文件夹不存在或某模块为空，不编造，跳过个性化直接生成通用话术
- 不要读出完整的个人信息原文塞进话术——提炼关键点，自然融入
- 敏感信息（手机号、真实姓名、具体公司名）不要直接写进话术，用 [手机号] 等占位

### Step 3：结合 JD 定位匹配点

如果对话历史中有 JD，从 JD 中找到 1-2 个和用户背景的匹配点。例如：JD 要求 Python → 用户项目里有 Python 数据处理项目 → "我看到贵司在招偏数据方向的工程师，我之前做 XX 项目正好是这块..."

**融入原则**：不做作、不报菜名、点到为止。一句话带过即可，不要罗列。

### Step 4：生成 3 版本话术 + 策略说明

每个场景生成 3 个版本，标签根据场景和态度动态选择（非固定）：

**常用标签组合：**
- 约面试 → 标签：偏主动/偏稳妥/偏简洁
- 问薪资 → 标签：自信报价/范围试探/反客为主（先问对方范围）
- 问离职/动机 → 标签：职业发展叙事/项目匹配叙事/简洁版
- 问空窗期 → 标签：学习成长型/项目填充型/简洁版
- 要材料 → 标签：积极展示型/精准匹配型/简洁版
- 面试确认 → 标签：正式确认型/灵活协商型
- Offer 沟通 → 标签：自信谈判型/稳妥接受型/争取时间型
- 通用/婉拒 → 标签：保持关系型/干脆利落型/追问反馈型

## 输出格式

```markdown
## 场景：[X] | HR态度：[Y]

### 版本 A — [标签]
[话术正文]

> 策略：[1-2句说明为什么这样回、适合什么情况]

### 版本 B — [标签]
[话术正文]

> 策略：[...]

### 版本 C — [标签]
[话术正文]

> 策略：[...]
```

## 八类场景详解

### 场景 1：约面试

**HR 典型消息：**
- "你什么时候方便面试？"
- "下周有空聊聊吗？"
- "方便电话沟通一下吗？"
- "你看什么时间方便？"

**关键信息需求：** 需要知道用户的时间可用性、所在城市（是否远程）

**规避点：**
- 不要第一次回复就说"我随时都可以"——显得太闲/太急
- 不要说"你们什么时候方便"——把球踢回去了，HR 要你给时间

**示例：**

> 假设用户是在校生，期望城市深圳，时间灵活。

#### 版本 A — 偏主动

> 下周整体比较方便，周一到周五下午都可以。我目前在 [城市]，如果方便的话可以线下面聊，远程也 OK。您看安排什么时间合适？

> 策略：给 HR 一个明确的窗口（下午），同时暗示可以线下——主动提供选项让 HR 觉得你好安排。适合 HR 比较主动热情的情况。

#### 版本 B — 偏稳妥

> 方便，下周下午的时间比较好安排。您看周三或周四下午方便吗？

> 策略：给 2 个具体时段，HR 选一个就行，减少来回确认的成本。适合中性态度的 HR。

#### 版本 C — 偏简洁

> 方便的~ 下周下午都可以，看您那边安排。

> 策略：简洁友好，用 "~" 显得轻松不拘谨。适合 HR 态度冷淡时——不卑不亢，简短但不冷漠。

---

### 场景 2：问薪资

**HR 典型消息：**
- "期望薪资多少？"
- "目前薪资方便说一下吗？"
- "我们这边薪资范围是 XX-XX，你能接受吗？"

**关键信息需求：** 需要用户的期望薪资范围（从个人信息读取，没有则不编造）

**规避点：**
- 不要直接报一个死数字——给区间
- 如果 HR 没给范围，可以先反问对方范围（反客为主）
- 不要说"薪资不重要"——自降身价

**示例：**

> 假设用户没有明确薪资记录在个人信息中，通用模板。

#### 版本 A — 自信报价

> 我的期望薪资在 [XX]K-[XX]K 之间，具体看岗位职责和成长空间。想了解一下贵司这个岗位的薪资结构是怎样的？底薪+绩效还是固定薪资？

> 策略：主动报区间 + 反问结构，展示专业度。适合有一定底气的候选人。

#### 版本 B — 反客为主

> 我这边对自己的薪资预期有一个大概范围，不过想先了解一下贵司这个岗位的预算范围？这样我们可以看是否在一个区间内，避免浪费彼此时间。

> 策略：不先亮底牌，礼貌反问。适合不确定市场价的候选人。

#### 版本 C — 范围试探

> 根据岗位要求和我的匹配度，期望在 [XX]K 左右。不过如果有其他福利（餐补/房补/期权），综合 package 可以酌情沟通。

> 策略：给出软性数字 + 留谈判空间。适合需要灵活性和安全感的情况。

---

### 场景 3：问离职/动机

**HR 典型消息：**
- "为什么离开上一家公司？"
- "为什么想来我们这里？"
- "目前有在看其他机会吗？"

**关键信息需求：** 用户的上一段经历（从 `工作经历.md`）、职业动机（从 `自我评价.md`）

**核心策略：** 永远不要贬低上家；用"职业发展需要"做框架；把原因引向"我要什么"而不是"我逃离什么"。

**示例：**

> 假设用户从上一家公司离开是因为想做更有技术挑战的项目。

#### 版本 A — 职业发展叙事

> 在上一家公司做了 [X 时间] 的 [岗位]，主要负责 [简要]。这段经历让我在 [技能] 上成长了很多，但公司目前的业务方向偏 [旧方向]，我个人更想在 [新方向] 深入。看到贵司在做 [JD 中的亮点]，觉得这个方向很匹配我的规划，所以很感兴趣。

> 策略：感谢过去 + 展望未来 + 锚定对方。这个结构在 HR 听来最舒服——你离开不是因为有问题，是因为有追求。适合大多数情况。

#### 版本 B — 项目匹配叙事

> 上一份工作让我积累了 [技能]，但做的项目偏 [形容]，想找一个能让我把这些经验用到 [新场景/新行业] 的机会。贵司的 [具体项目/方向] 正好是我特别想做的方向。

> 策略：把离职原因和应聘岗位直接挂钩——"我不是随便跳，我就是想跳到你这里"。适合目标明确的候选人。

#### 版本 C — 简洁版

> 主要是职业规划的原因。上一家公司很好，但我想在 [方向] 上走得更深一些，目前的方向不太匹配。在看机会，但比较挑，贵司是我特别感兴趣的一家。

> 策略：四两拨千斤——不展开但给足面子 + 轻拍对方马屁。适合 HR 随口一问的场景。

---

### 场景 4：问空窗期

**HR 典型消息：**
- "这段时间在做什么？"
- "为什么 Gap 了这么久？"
- "中间这段时间是怎么安排的？"

**关键信息需求：** 空窗期做了什么（从 `自我评价.md`、`项目列表.md` 找空窗期项目或学习）

**核心策略：** 把"空窗"翻译成"投入"。不管做了什么——学了什么、做了项目、考了证、休息调整——都讲成有意识的决定，不是被动等待。

**示例：**

> 假设用户在空窗期做了一个个人项目 + 系统补了一些技术。

#### 版本 A — 学习成长型

> 这段时间我花了不少时间在 [具体技术/项目] 上——做了一个 [项目名/项目方向]，主要是想把自己在 [技能] 这块的能力补扎实。中间也看了不少 [方向] 的深度内容。现在觉得准备得差不多了，开始认真看机会。

> 策略：展示自主性——不是"没工作闲着"而是"有目标地投入"。用具体的项目名/技术名增加可信度。适合空窗期确实有产出的情况。

#### 版本 B — 项目填充型

> 中间接了一个 [项目类型]，独立完成了 [简要成果]。所以也不是完全空着，只是没有正式入职。现在项目告一段落，想找一个稳定团队长期做。

> 策略：用项目把空窗期"填上"，让 HR 觉得你其实一直在做事。适合有自由职业/接项目经历的情况。

#### 版本 C — 简洁版

> 中间给自己留了一段时间——一方面是调整一下节奏，另一方面是想清楚下一步要往哪个方向走。现在方向明确了，开始认真看机会。

> 策略：诚实但不解释过度。用"有意识的决定"替代"尴尬的真空"。适合空窗期较短或确实在休息的情况。

### 场景 5：要材料

**HR 典型消息：**
- "发一份详细简历过来"
- "有没有相关项目可以看看？"
- "方便发一下作品集吗？"
- "有没有 GitHub/个人网站？"

**关键信息需求：** 用户的简历文件路径、项目列表（从 `项目列表.md`）、是否有现成作品集或 GitHub

**规避点：**
- 不要只回复"好的已发"——趁机用一个钩子让 HR 打开看
- 如果项目不匹配，诚实说，不要硬凑

**示例：**

> 假设用户简历已生成，有 2 个和 JD 相关的项目。

#### 版本 A — 积极展示型

> 好的，简历已发~ 其中 [项目A] 和 [项目B] 跟贵司这个岗位方向比较匹配。[项目A] 里我主要负责 [一句话亮点]。您先看看，有什么想详细了解的随时问我。

> 策略：发材料的同时抛一个钩子——点名 2 个项目 + 亮点引导 HR 去关注。HR 大概率会顺着你点的方向看。

#### 版本 B — 精准匹配型

> 已发~ 附了详细简历和一个跟 [JD方向] 相关的项目说明。您可以重点看一下 [具体章节/项目]。

> 策略：直接告诉 HR 看哪里，减少 HR 的阅读成本。适合 HR 明确要特定材料的场景。

#### 版本 C — 简洁版

> 好的，简历和作品集都发您了。有任何需要补充的随时和我说~

> 策略：干净利落。适合 HR 只是走流程要材料、没有太多互动意愿的场景。

### 场景 6：面试确认

**HR 典型消息：**
- "已安排周三下午 3 点面试，请确认"
- "时间是 X 日 X 点，方便吗？"
- "面试链接：XXX，确认一下"

**关键信息需求：** 用户的时间可用性（从 `基本信息.md`）

**核心策略：** 如果时间 OK，确认 + 简短表达期待；如果时间不行，立刻给替代时间，不要只说"不行"。

**示例：**

#### 版本 A — 正式确认型（时间 OK）

> 收到，周三下午 3 点没问题。我会提前准备好，准时参加。谢谢您~

> 策略：确认 + 给人靠谱的印象。适合大多数面试确认场景。

#### 版本 B — 灵活协商型（时间不行）

> 抱歉，周三下午有点冲突。周四或周五同一时间方便吗？或者周三上午 10 点-12 点也行。给您带来不便了~

> 策略：不行的同时马上给替代方案，不给 HR 制造额外的调度思考成本。

---

### 场景 7：Offer 沟通

**HR 典型消息：**
- "我们给到的薪资是 XX，你觉得怎么样？"
- "offer 详情已发，期待你的回复"
- "还有哪些方面想了解的？"
- （用户想主动谈薪资/福利）

**关键信息需求：** 用户的期望薪资、期望城市、市场行情（从 `基本信息.md`、`工作经历.md`）

**规避点：**
- 不要在第一次沟通就说"可以"——给自己留时间
- 不要一上来就砍价——先感谢再谈
- 不确定时就说"我考虑一下，X 日前回复"，不要沉默

**示例：**

> 假设用户拿到一个跟期望接近但略低的 offer。

#### 版本 A — 自信谈判型

> 谢谢 offer~ 整体很感兴趣，团队和方向都很匹配。不过薪资方面，我希望能在 [X]K 上下，比我目前的预期略低了一点。如果这个数字有空间的话，我这边基本上可以确定。您方便帮忙沟通一下吗？

> 策略：先正面反馈（降低 HR 的防御）→ 再提诉求 → 用"基本上可以确定"给 HR 谈判动力。适合手上有其他选择或对自己估值有信心的候选人。

#### 版本 B — 稳妥接受型

> 谢谢！薪资和整体 package 我看了，在接受范围内。我这边确认接受。后续入职流程需要我准备什么？

> 策略：干脆确定，展示职业素养。适合薪资符合预期、不想再拉扯的情况。

#### 版本 C — 争取时间型

> 非常感谢~ 整体 package 我仔细看一下，明天之内给您确认回复。另外还有一个小问题想了解：[福利/公积金/试用期相关] 这个方便说一下吗？

> 策略：不马上答应但给出明确的回复时间（不是无限期吊着），顺便问一个实际关心的细节。适合需要时间考虑或对比的情况。

### 场景 8：通用/婉拒

**子场景 — 8a：HR 说"不太合适"**
- 保持关系："感谢您的反馈，如果后续有更匹配的机会欢迎联系"
- 追问反馈："方便简单说一下是哪方面不太匹配吗？对我后续调整也有帮助"
- 转移方向："贵司还有其他方向在招吗？[XX方向] 我也可以考虑"

**子场景 — 8b：你想拒绝 Offer/面试**
- 留后路型："经过仔细考虑，这次可能不是最好的时机。非常希望以后有机会再合作"
- 干脆利落型："感谢您的时间和 offer，我决定先 pursue 另一个机会。祝早日找到合适的人选"
- 推荐他人型（如适用）："我个人这次不太合适，但我认识一个 [方向] 的朋友可能感兴趣..."

**子场景 — 8c：催面试进度**
- HR 长时间没回复，礼貌跟进
- "您好~ 上次聊到 [话题]，想问一下后续进展如何？"
- 不要用"你们是不是不要我了"之类的话术——永远保持专业

**子场景 — 8d：收到拒信后回复**
- 礼貌回复，保持关系网："感谢告知~ 方便的话想了解一下，如果后续有更匹配的方向，希望能有机会再聊。"

## 话术原则

### ✅ DO

- **口语化**：像真人微信聊天，用 适当语气词，不过度使用用书面语
- **有钩子**：每条回复给 HR 一个继续对话的理由——问问题、抛信息点、给时间承诺
- **见人下菜**：HR 急则准、HR 热则温、HR 冷则钩（用信息点拉回对话）
- **个人信息自然融入**：不刻意报菜名，只提和这个岗位相关的
- **留空给用户填**：话术里用 `[时间]`、`[X]K`、`[项目名]` 等占位，然后告诉用户填什么
- **篇幅适中**：微信聊天一段不超过 3-4 行。多用换行分段
- **保持专业**：即使用户被拒绝或遇到粗暴 HR，话术始终保持专业和体面

### ❌ DON'T

- 不要用书面化的长句——"鉴于贵司的业务发展方向与本人的职业规划高度契合"
- 不要一次回复两个以上的大问题——HR 会有压力
- 不要把个人信息里的敏感数据（手机号、真实姓名、公司名）直接塞进话术
- 不要虚构经历或技能来迎合 HR
- 不要替用户做决定（如"我接受 offer"）——给话术，让用户选
- 不要贬低上家、同事、行业
- 不要用"你们公司怎么怎么样"——换成"贵司"

## 话术交付后的提示

生成话术后，告知用户：

1. `[方括号]` 里的内容需要根据实际情况填写
2. 选一个最符合你当前心态的版本微调即可
3. 如果 HR 的回复和预期不同，把新消息贴过来，再生成下一轮话术

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 话术像商务邮件 | 想象你在微信上跟一个不太熟的人聊天 |
| 一次回复太长 | 控制单条消息在手机屏幕可见范围内 |
| 三个版本差不多，只是换了词 | 三个版本必须有明确的策略差异（姿态/信息量/钩子方式不同） |
| 没读个人信息就生成 | 先读相关部分，没有则声明"未找到 XX 信息，用通用模板" |
| 给 HR 设置 Deadline 式回复 | 永远不要代用户说"我明天之内回复你"——你替用户把话说出去了，用户被动 |
| 忽略 HR 态度，统一语气 | HR 冷你暖是尬聊，HR 热你冷是傲慢。必须匹配 |
| 场景判断错了 | 如果不确定 HR 意图，优先当成"约面试"场景处理，同时用最开放的话术 |
