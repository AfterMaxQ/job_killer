# 常见错误与自检清单

> 供 build-resume SKILL.md 编译前/后引用。Agent 在提交前 Read 此文件逐一核对。

## Agent 自检清单

Agent 在两个关键时刻必须逐项核对：

### Markdown 定稿前（阶段四结束时）

- [ ] 每个项目内容是否说明了具体痛点？（谁有痛点 → 现有方案缺陷 → 本方案怎么解决）
- [ ] 每个项目成果 ≥2 个分句？（落地状态 + 全流程能力/可复用价值）
- [ ] 所有技能是否来自 `个人信息/` 或用户明确确认？每行是否 ≤5 个技术名词？是否合并了重合度高的同类技术？是否混入了 API 基础调用方式？
- [ ] 是否删除了所有「精通」「熟练掌握」等无数据支撑的修饰词？用户要求熟练度时是否只用「熟练」「掌握」？
- [ ] 个人信息字段是否正确？（居住地 vs 籍贯、邮箱 label 对应正确）
- [ ] 总字数在 800-1000 字范围内？

### LaTeX 组装前（阶段七 7.1）

- [ ] 所有独立 `{...}` 组之间是否加了 `\par`？（技术栈↔项目内容、GPA↔核心课程、技能各行之间）
- [ ] 每条 `\Content` 要点 ≤60 字？
- [ ] 正文描述中无 `\textbf{}`？（仅板块标题、项目名称可加粗；分类前缀词使用 `\prefix{}`）
- [ ] 分类前缀词已统一使用 `\prefix{}`？（技术栈/项目内容/项目成果/核心课程/竞赛获奖/技能类别等）

### 编译后（阶段七 7.3）

- [ ] `ls *.cls *.sty *.aux *.log *.out` 无残留文件？
- [ ] `resume.pdf` 确认为 1 页？
- [ ] 已提醒用户打开 PDF 逐页检查对齐、溢出、间距？

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 不读取 temp_resume/ 就直接询问用户 | 优先从 temp_resume/ 读取已有内容，只问缺失的 |
| 项目要点超过 60 字或缺乏动词开头 | 每点控制 60 字内，以动词开头，遵循「动作+技术+场景+结果」 |
| 个人信息中用户没有的字段仍然保留空行 | 直接删除没有的字段对应的 LaTeX 行 |
| 编译失败不展示错误信息 | 展示 xelatex 最后 15 行 |
| 跳过环境检查直接编译 | 必须先检查 xelatex 可用性，不可用时引导安装 |
| 编造用户没有的经历或数据 | 所有内容必须有 temp_resume/ 或 个人信息/ 来源 |
| 日期格式混用 | 统一 `YYYY.M` 格式 |
| `\Content` 要点超出 60 字导致溢出 | 严格数字数，技术术语用 `\mbox{}` |
| 正文描述使用 `\textbf{}` 加粗 | 仅板块标题、项目/竞赛名称可加粗；分类前缀词使用 `\prefix{}` |
| 分类前缀词未使用 `\prefix{}` | 前缀词统一用 `\prefix{技术栈：}`，大一号加粗，与内容形成层次 |
| PDF 内容不足一页或刚好超出一页 | 调整内容或间距填满一页 |
| 编译后不提醒打开 PDF 复核 | 必须提醒逐页检查 |
| 硬编码 resume.tex 内容 | 必须从模板动态组装，按 temp_resume/ 模块选择 |
| Markdown→LaTeX 阶段重问排版决策 | 排版决策在阶段三确定，后续只读取不重问 |
| 项目内容只描述功能不写痛点 | 遵循「痛点写作三步法」：谁有痛点→现有方案缺陷→你的方案怎么解决 |
| 项目成果只有 1 句话 | 至少 2 个分句，覆盖落地状态 + 全流程能力/可复用价值 |
| 连续 `{...}` 组不加 `\par` 导致文字连行 | 技术栈与项目内容之间、GPA 与核心课程之间等所有独立 `{...}` 段落之间加 `\par` |
| 编造用户没有的技能或经历 | 只使用个人信息/ 和用户确认过的内容，不确定就问，不推测 |
| 部署环境/技术细节与用户实际不符 | 从个人信息/ 读取实际环境（Mac/Linux/Windows），不擅自改写 |
| 技能名称加括号注解（如 `JWT（HS512）`） | 去掉括号，只写技术名称。版本号、配置细节等留给面试展开 |
| 技术岗自我评价写商务英语/翻译/跨文化沟通 | 自我评价紧扣岗位类型，技术岗写技术热情+执行闭环+学习能力，语言能力放教育背景简要提及即可 |
| 形容词堆砌（"精通""熟练掌握"） | 去修饰词，用具体技术栈名称和数据替代 |
| 专业技能罗列 10+ 个重合度高的技术 | 每行 ≤5 个，只挑最核心、JD 明确要求的，合并同类项 |
| 将 API 基础调用方式列为技能 | 调包不算技能 |
| 技能加"精通"修饰词 | 默认不加熟练度。用户要求时只用「熟练」「掌握」，禁止「精通」 |
| 个人信息字段用错（籍贯 vs 居住地） | 严格按用户选择展示，首次确认后将决策记录到排版配置中 |
