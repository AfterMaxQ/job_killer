# 个人信息初始化

## 概述

在项目根目录下创建标准化的 `个人信息/` 文件夹结构，按简历模块分类存放用户的个人信息、技能、经历、评价等内容。所有信息以 .md 格式存储，便于后续 agent 检索和调取。

## 适用场景

- 首次搭建个人简历档案库
- 项目根目录下没有 `个人信息/` 文件夹
- 用户希望将零散的个人信息系统化整理
- 用户说"帮我初始化"、"整理我的资料"、"建立档案"

## 文件夹结构

```
个人信息/
├── 个人基本信息/           # 身份信息
│   └── 基本信息.md          # 姓名、年龄、联系方式、求职意向等
├── 个人专业技能/           # 技术 & 软技能
│   ├── 编程技术.md          # 编程语言、框架、工具
│   ├── 语言能力.md          # 外语水平、证书
│   └── 其他能力.md          # AI工具、DevOps等
├── 个人项目经历/           # 项目经验
│   └── 项目列表.md          # 各项目的描述、技术栈、成果
├── 个人工作经历/           # 实习 & 工作
│   └── 工作经历.md          # 公司、职位、职责、成果
├── 个人比赛经历/           # 竞赛 & 比赛
│   └── 比赛经历.md          # 比赛名称、时间、成绩、角色、工作与成果
└── 个人自我评价/           # 自评 & 亮点
    └── 自我评价.md          # 核心优势、性格特质、职业目标
```

## 工作流程

### 第一步：检查现有状态

1. 检查项目根目录是否已有 `个人信息/` 文件夹 或是 xx信息 等像是信息档案的文件
2. 如果已存在，列出当前目录结构，询问用户是否需要更新，如果要更新要改哪个模块
3. 如果不存在，进入初始化流程

### 第二步：创建目录结构

一次性创建所有文件夹：

```
个人信息/
个人基本信息/
个人专业技能/
个人项目经历/
个人工作经历/
个人比赛经历/
个人自我评价/
```

### 第三步：引导用户提供信息

按优先级逐模块收集信息。每次只问一个模块，避免信息过载。

#### 模块 1：个人基本信息

询问并记录：

- 姓名
- 联系电话
- 邮箱
- 出生年份 / 年龄
- 出生地
- 当前居住地
- 当前身份（如：XX大学 本科在读）
- 期望工作城市（可多个）
- 期望行业（可多个）
- 期望岗位方向（可多个）
- 其他身份标识（如：辅修专业、特殊身份）

存储为 `个人信息/个人基本信息/基本信息.md`，格式：

```markdown
# 个人基本信息

## 身份信息
- 姓名：
- 联系电话：
- 邮箱：
- 出生年份：
- 出生地：
- 当前居住地：
- 当前身份：

## 求职意向
- 期望工作城市：
- 期望行业：
- 期望岗位方向：

## 其他
- （其他补充信息）
```

#### 模块 2：个人专业技能

询问并记录：

- 编程语言及熟练度
- 框架 & 库
- 数据库 & 中间件
- AI/ML 相关技能
- 开发工具 & DevOps
- 其他技术能力

存储为 `个人信息/个人专业技能/编程技术.md`，按技术类别分段。

语言能力单独存储为 `个人信息/个人专业技能/语言能力.md`：
- 语种及水平（如：英语 CET-6、粤语母语）
- 证书
- 实际使用场景

其他工具能力（AI 编码工具、设计工具等）存储为 `个人信息/个人专业技能/其他能力.md`。

#### 模块 3：个人项目经历

询问并记录每个项目：

- 项目名称
- 项目时间
- 项目角色
- 技术栈
- 项目背景/目标
- 核心职责与成果
- 量化指标（如有）

存储为 `个人信息/个人项目经历/项目列表.md`，每个项目一个章节。

#### 模块 4：个人工作经历

询问并记录每段经历：

- 公司/机构名称
- 时间段
- 职位
- 核心职责
- 关键成果

存储为 `个人信息/个人工作经历/工作经历.md`。

#### 模块 5：个人比赛经历

询问并记录每场比赛：

- 比赛名称
- 比赛时间
- 比赛成绩/奖项
- 核心角色（如：队长、核心成员、独立参赛）
- 项目/赛题名称
- 核心工作与成果（逐条列出，尽量量化）
- 关键发现/结论/价值
- 技术栈/方法论（如有）

存储为 `个人信息/个人比赛经历/比赛经历.md`，每场比赛一个章节。

#### 模块 6：个人自我评价

询问并记录：

- 核心优势（3-5 条）
- 性格特质
- 职业目标
- 独特亮点（跨领域能力、特殊经历等）

存储为 `个人信息/个人自我评价/自我评价.md`。

### 第四步：确认与验证

1. 列出已创建的所有文件和文件夹
2. 请用户确认信息是否准确、完整
3. 如有遗漏，补充后再确认

## 信息收集原则

### 必须遵守

- **不编造信息**：只记录用户明确提供的内容
- **不遗漏信息**：用户提供的每一条都记录，不自行筛选
- **保持原意**：可优化表述使其通顺，但核心事实不变
- **结构化优先**：用列表、表格等结构化方式组织，便于程序化读取

### 文件命名规范

- 使用中文文件名，直观易读
- .md 格式，UTF-8 编码
- 文件名简洁：`基本信息.md`、`编程技术.md`、`项目列表.md`

### 内容格式规范

- 使用 Markdown 标题层级组织信息
- 列表项以 `-` 开头
- 键值对使用 `- 键：值` 格式
- 关键数字和日期保持原值

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 跳过确认步骤，初始化完就结束 | 必须请用户验证所有信息正确 |
| 一次性问用户所有模块的问题 | 逐模块引导，每次只问一个模块 |
| 自行修改用户提供的数据 | 只能优化表述，不能改变事实 |
| 文件夹创建后不创建初始 .md 文件 | 每个文件夹至少有一个占位或初始化文件 |

## 初始化完成后的提示

初始化完成后，告知用户：

1. 文件夹结构已建立，路径为 `项目根目录/个人信息/`
2. 已存储的模块清单
3. 如何使用：后续使用 build-resume 生成简历或 greeting-generator 生成招呼语时，Agent 会自动读取这些文件
4. 如何更新：随时说"更新个人信息/XX模块"即可
